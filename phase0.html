    Phase 0: Planned Database Schema

    These collections define the MongoDB databases planned during Week 2. You will implement these in Phase 1.

    Users Collection Schema 
    Users Collection:
- _id (unique ID generated by MongoDB)
- name (string, required)
- email (string, unique, required)
- passwordHash (string, hashed using bcryptjs)
- role (enum: 'elderly', 'family', 'caregiver', 'admin')
- phone (string)
- profilePicture (string URL)
- createdAt (timestamp)
    HealthRecords Collection Schema 
    HealthRecords Collection:
- _id (unique ID)
- elderlyUserId (reference to Users._id)
- bloodPressure (object: { systolic: number, diastolic: number })
- bloodSugar (number, mg/dl)
- heartRate (number, bpm)
- bodyTemperature (number, °F)
- weight (number, lbs)
- mood (number, 1-10 scale)
- sleepHours (number)
- notes (string, caregiver observations)
- recordedBy (reference to Users._id caregiver)
- createdAt (timestamp)
    Emergency Collection Schema 
    Emergency Collection:
- _id (unique ID)
- elderlyUserId (reference to Users._id)
- status (enum: 'active', 'resolved', 'false_alarm')
- latitude (number, coordinates)
- longitude (number, coordinates)
- locationUpdatedAt (timestamp)
- description (string)
- triggeredAt (timestamp)
- resolvedAt (timestamp)
- notifiedContacts (array of reference Users._id)
- severity (enum: 'low', 'medium', 'high')
