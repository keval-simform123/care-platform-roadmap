    What YOU Must Audit

    Do not copy-paste code for these features blindly. Manually test and double-check all logic.

    • ⚠️ EmergencySOS pipelines: Verify SOS location triggers log correctly on database levels before dispatching network sockets.
    • ⚠️ Role-Based security check: Inspect backend authorization headers (`roleCheck.js`) to block data leaks between unrelated users.
    • ⚠️ Token decryption: Audit how the local DataStore caches token values, and verify tokens wipe clean on logging out.
    • ⚠️ Unit Testing & Validation: Write test suites in Postman to challenge permission boundaries with negative assertions.

    Deployment Playbook

    Follow these structured deployment checklists when preparing the platform for public release.

    1. Deploy Express Server Backend API
    Purchase target domain name (e.g. careplatform.in) and upload backend folders to cloud platforms like DigitalOcean, AWS, or Heroku. Setup SSL certificates (using Let's Encrypt) to enforce secure HTTPS channels.
    2. Configure Production Database MongoDB Atlas
    Transition database from sandbox cluster to dedicated Atlas networks. Setup database backup protocols (daily/weekly snapshots) and whitelist API production IP ranges in security tabs.
    3. Sign Android APK Keystore Android Client
    Generate release keystores in Android Studio and build the production-ready signed APK bundle. Replace base API URLs inside client configurations to target production routes.
    4. Deploy to Google Play Console Public Release
    Register developer accounts, fill out policy checks, upload app icons/screenshots, attach target privacy policies, and submit signed APK bundles. Review cycles generally complete in 24-48 hours.
