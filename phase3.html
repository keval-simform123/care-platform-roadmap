    Manual Testing Cases (Weeks 15-16)

    These core test matrices should be executed manually before beta user release.

    Test Case Execution Step Expected Behavior
    Valid Registration Submit registration form with unique email and valid passwords. API returns 201 Created; passwords appear fully hashed in database.
    Duplicated Email Checks Attempt registration with an email address already registered. API intercepts, preventing duplication, and returns 400 Bad Request.
    Invalid Credentials Login Input incorrect passwords for an existing account and click login. Backend denies access, returning 401 Unauthorized.
    Expired Token Validation Attempt accessing dashboard routes with a stale or expired JWT token. API intercepts, returning 401 Unauthorized, and routes client back to login screen.
    Test Case Execution Step Expected Behavior
    Cross-Family Data Leak Blocks Request vitals of Elderly relative A using Family member B's token. Middleware halts execution, returning 403 Forbidden. Access is blocked.
    Caregiver Scope Validation Attempt accessing client records of an unassigned patient. Middleware validates relationship maps and yields 403 Forbidden.
    Unassigned Chat Blocks Dispatch a message object between users with no established links. API verifies relationships, blocking dispatch, and returns 403.
    Test Case Execution Step Expected Behavior
    SOS Database Persistence Tap the red SOS button and trigger the emergency handler. Active SOS record is logged with coordinates before push notices trigger.
    Dynamic Location Tracking Move client device while emergency is active. Latitude/Longitude coordinates update every 10s on the tracking database.
    Emergency Resolution Tap resolve button from linked family member panel. SOS status updates to 'resolved' and family maps are updated.
