    Phase 1: Backend Development

    Assemble the database infrastructure, core express controllers, and configure socket networks.

    Duration: Month 1 (Weeks 3-6)
    Goal: Active API Deployment

    Phase 1 Week-by-Week Execution

    Boilerplate dependencies

    Run these npm packages during initial setup:

    npm install express mongoose bcryptjs jsonwebtoken cors dotenv

    Relational Integrity

    Mongoose references are critical. Always index reference keys (e.g. `{ elderlyUserId: 1 }`) to ensure dashboard load times under 2 seconds at scale.

    Emergency Priority

    Persist SOS logs to MongoDB immediately before attempting to dispatch push notifications. An Atlas connection crash must not block the core emergency trace.

    Phase 1 Completion Checklist

    • JWT Auth validates correctly
    • Role-Based security verified
    • SOS updates coordinates
    • Endpoints tested in Postman

    API Endpoint Reference Directory

    Quickly search or filter planned API endpoints. Use the search bar to locate routes.

    Method Endpoint Access Level Functional Goal
    POST /auth/register Public Register user (role: elderly, family, caregiver, admin)
    POST /auth/login Public Authenticate password, return token & user profile details
    GET /elderly/:id/health-records Linked Roles Get chronological health logs list for dashboard trends
    POST /elderly/:id/health-records Caregiver / Admin Log new blood pressure, glucose, temperature, and mood metrics
    POST /emergency/sos Elderly Only Activate SOS status, log GPS, and trigger notification array
    POST /emergency/:id/update-location Elderly Only Track coordinate shifts during active emergency cycles
    POST /messages/send Linked Roles Send real-time encrypted texts or GPS sharing alerts
    GET /reports/health-summary/:elderlyId Linked Roles Compile monthly vitals and medical compliance charts
    GET /admin/statistics Admin Only Retrieve total active emergencies and system node performance stats
