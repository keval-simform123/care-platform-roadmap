    Encrypted DataStore

    Do not utilize standard SharedPreferences for storing login tokens. Standard XML files are unencrypted on rooted phones. Always use Preference DataStore or EncryptedSharedPreferences.

    Accessibility details

    Elderly dashboards must utilize:
    • Font sizes of at least 16sp
    • High-contrast backgrounds
    • Touch targets of at least 48dp height

    Vitals Charts integration

    Use standard libraries like PhilJay's MPAndroidChart in your dependencies for simple line charts, or compose-based Canvas drawings to prevent memory leaks on old tablets.

    FCM Channels checklist

    • High importance for SOS alert channels
    • Default sound for message alerts
    • Request runtime permissions on Android 13+

    Mobile-to-Backend Architecture Flow

    How the Kotlin client connects securely to your deployed Express APIs.

    1. Authentication Step: User enters details on Login Screen. App calls POST /auth/login.
    2. Token Storage: On success, JSON response returns a JWT string. App caches it in DataStore.
    3. OkHttp Interceptor: Subsequent API queries pass through an interceptor inserting the cached token:
      Authorization: Bearer {JWT_TOKEN}.
    4. Dynamic Navigation: Home screen loads. Retrofit fires GET requests based on stored user roles (e.g. /elderly/me).
